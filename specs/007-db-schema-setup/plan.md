# Implementation Plan: Database & Configuration Setup

**Branch**: `007-db-schema-setup` | **Date**: 2026-03-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-db-schema-setup/spec.md`

## Summary

Finalize database schema for Becslo pricing calculator, seed initial data (services, categories, countries, costs), configure row-level security for user data isolation, and implement API endpoints for wizard data access. This phase follows Phase 0 (authentication) and enables Phase 2 (pricing engine) and Phase 3 (wizard flow).

## Technical Context

**Language/Version**: TypeScript (NextJS 14.x)  
**Primary Dependencies**: NextJS 14.x, Supabase (auth, database), Supabase migrations  
**Storage**: Supabase PostgreSQL  
**Testing**: Jest, Playwright  
**Target Platform**: Web (Vercel hosting)  
**Project Type**: Web application (monolithic NextJS + Supabase)  
**Performance Goals**: API responses <500ms, seed data loads <2s  
**Constraints**: USD currency only, nearest dollar precision, RLS for user isolation, admin full access  
**Scale/Scope**: Single-tenant per user, admin manages all data, 120+ services, 200+ countries

### 1.1 Data Model

**Output**: `data-model.md`

- Cost (optional overhead)
- Config (system settings)

### 1.2 Interface Contracts

**Output**: `/contracts/api-contracts.md`

Five API endpoints:
- GET `/api/v1/services` - Fetch services (filtered by category, active only)
- GET `/api/v1/categories` - Fetch all categories
- GET `/api/v1/countries` - Fetch all countries with multipliers
- GET `/api/v1/costs` - Fetch active costs
- GET `/api/v1/config` - Fetch system configuration
