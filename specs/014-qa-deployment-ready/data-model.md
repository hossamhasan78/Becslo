# Data Model: Phase 7 Deployment & Verification

## Entity: Test Report
Represents a snapshot of a QA/Test run results, typically generated by the CI/CD pipeline or during manual production verification.

| Field | Type | Description |
| :--- | :--- | :--- |
| **id** | UUID | Unique identifier for the test run. |
| **timestamp** | DateTime | When the test run started. |
| **environment** | Enum | Valid values: `preview`, `production`, `local`. |
| **status** | Enum | Valid values: `PASSED`, `FAILED`, `CANCELLED`. |
| **version** | String | Git SHA or Build ID being tested. |
| **result_summary** | String | A brief overview of which test suites (e.g., `AuthP1`, `WizardP1`) passed. |
| **report_url** | String | Link to the full Playwright or Vitest HTML report. |

## Entity: Environment Configuration
Defines the connection and security parameters for different deployment targets.

| Field | Type | Description |
| :--- | :--- | :--- |
| **id** | String | Friendly name (e.g., `Vercel-Prod`). |
| **target_env** | Enum | `PREVIEW` or `PRODUCTION`. |
| **base_url** | String | The root URL for testing. |
| **supabase_url** | String | Target database URL. |
| **auth_provider** | String | Always `Supabase-Auth` for MVP. |

## Relationships
- A **Test Report** is strictly associated with a specific **Environment Configuration** and a source code **Version**.
- **Admin documentation** (though not an entity in DB) relates to the specific **Environment Configuration** for user onboarding.
